# Home Page UI 优化规格

## 背景与目标

- 当前 Home 页面功能完整，但视觉上存在“按钮、标题、列表混排”的割裂感，分区层级不够清晰。
- 本次优化目标是在**不改变业务能力与交互结果**的前提下，提升页面的简洁度、规整度和风格一致性。
- 优化范围仅限 Home 页面 UI 表现与样式组织，不新增业务流程，不移除已有入口。

## 现状问题

- 关键入口（Recent/Favorites）与目录列表（Local/SD/Network）视觉权重接近，导致主次不明。
- 区块间距与内部间距节奏不统一，页面纵向扫描效率低。
- 列表项样式依赖系统默认与自定义灰底混用，颜色、圆角、文本层级不一致。
- 部分文本与弹窗文案仍为硬编码英文，不利于整体风格统一与后续国际化。

## 设计原则

- 功能不变：保留所有现有入口、点击行为、权限流程、数据绑定逻辑。
- 结构清晰：按“快捷操作 / 存储来源 / 网络来源”三层组织页面信息。
- 视觉统一：统一卡片容器、列表项样式、标题层级、间距系统。
- 主题适配：优先使用主题属性与资源，避免硬编码颜色，保证浅色/深色一致性。

## 范围说明

### 包含
- `fragment_home.xml` 的页面结构重排与样式统一。
- Home 页面所需新增或调整的 UI 资源（shape、dimens、strings）。
- Home 页面列表项布局统一（本地/SD/网络采用一致 item 样式）。
- Home 页面按钮样式统一（Recent/Favorites 统一层级和间距）。

### 不包含
- Home 页面功能逻辑改写（如网络配置保存逻辑、目录跳转逻辑、权限机制）。
- 其它页面（Files/Reader/More/Favorites/Recents）的视觉改造。
- 导航结构调整（底部导航项与页面路由保持不变）。

## 信息架构与布局规范

### 页面结构
- 顶层继续使用可滚动容器，防止小屏截断。
- 分为三个区块：
  - 快捷入口区：Recent Opens、Favorites 两个主要操作。
  - 本地来源区：Local Directories、SD Card。
  - 网络来源区：Network Connections（含添加入口）。

### 区块规则
- 每个区块使用统一卡片式容器（圆角 + 轻量边界/背景）。
- 区块标题使用同一文本样式（字号、字重、颜色）。
- 区块内部元素采用统一垂直间距 token（如 8/12/16dp 阶梯）。

### 列表与交互项
- 三个 ListView 使用统一 item 布局，而非系统默认 item。
- 列表项高度、左右内边距、文本字号统一；超长文本单行省略。
- 列表点击态反馈一致（ripple 或 selector），增强“可点击”感知。

## 资源与样式约束

- 将 Home 相关间距抽取到 `dimens.xml`（避免 magic number）。
- 将 Home 相关文本补齐到 `strings.xml`，移除新增硬编码文案。
- 新增 Home 区块容器与列表项背景资源，统一命名前缀（如 `home_`）。
- 保持与现有主题体系兼容，优先复用 `?attr/colorSurface`、`?attr/colorOnSurface` 等语义色。

## 功能保持清单（必须保持）

- Recent 按钮仍跳转最近历史页面。
- Favorites 按钮仍跳转收藏页面。
- Local 列表仍默认可进入 Downloads。
- SD 列表仍支持“Add SD Card (SAF)”并触发目录授权选择器。
- Network 列表仍支持“新增连接”入口与已有连接跳转。
- 网络配置实时观察（Flow collect）与刷新逻辑保持不变。
- Android 11+ 全文件权限提示逻辑保持现有行为。

## 兼容性与风险

- 兼容 Android 11+ 当前最低支持版本，不引入新依赖。
- 保留 ListView 方案，不强制迁移 RecyclerView，控制改动风险。
- 需重点验证不同字体缩放下的文本截断与区块高度表现。

## 验收标准

- Home 页面功能行为与优化前一致，无入口丢失。
- 页面视觉主次清晰：快捷入口、来源区块可快速识别。
- 间距、字体层级、列表项样式在三个区块中保持统一。
- 浅色/深色主题下均可读、无明显颜色冲突。
- 文案资源化后不出现新增硬编码文本。
