# 提升 Folders 文件浏览效率 Spec

## Why
Folders 页面在文件数量较多时，浏览成本高、定位目标文件慢。需要补充排序、字体大小、快速滑动与信息密度控制能力，提升大目录下的可用性。

## What Changes
- 增加“按文件大小升序”排序选项（与现有排序体系并存）。
- 增加文件名字体大小档位设置（小/中/大，默认中）。
- 增加侧边快速滑动滑块，支持拖拽快速定位列表位置。
- 限制文件名主展示最多两行，超出省略；属性信息（大小/时间）不计入该两行限制。
- 将类型标记从 `[FILE]` / `[DIR]` 简化为 `[F]` / `[D]`。

## Impact
- Affected specs: 目录浏览与列表可读性、长列表交互效率
- Affected code: `FilesFragment` 列表渲染与排序选项、相关布局资源、可选持久化配置（如 SharedPreferences）

## ADDED Requirements
### Requirement: 按大小升序排序
系统 SHALL 提供“按大小升序”排序选项，并在切换后立即对当前列表生效。

#### Scenario: 用户切换为按大小升序
- **WHEN** 用户在 Folders 页面选择“按大小升序”
- **THEN** 文件项按 size 从小到大排序
- **AND** 目录项排序行为保持与现有规则一致（不破坏目录可浏览性）

### Requirement: 文件名字体大小档位
系统 SHALL 提供文件名字体大小 3 档（小/中/大），默认使用中档。

#### Scenario: 用户切换字体档位
- **WHEN** 用户将文件名字体从中档切换到小档或大档
- **THEN** 当前列表立即更新字体大小
- **AND** 再次进入页面时保持上次选择

### Requirement: 侧边快速滑动
系统 SHALL 在长列表场景提供侧边拖拽滑块用于快速滚动。

#### Scenario: 拖拽快速滑块
- **WHEN** 用户拖拽侧边滑块
- **THEN** 列表滚动位置随拖拽比例快速变化
- **AND** 不影响普通手势滚动

### Requirement: 文件名最多两行展示
系统 SHALL 将文件名主展示限制为最多两行，超出内容省略。

#### Scenario: 超长文件名
- **WHEN** 文件名长度超过可展示宽度
- **THEN** 文件名主区域最多显示两行并使用省略
- **AND** 属性行仍独立显示，不与文件名行数限制互相挤占

### Requirement: 类型标记简化
系统 SHALL 将条目前缀类型标记简化为 `[F]`（文件）和 `[D]`（目录）。

#### Scenario: 渲染列表项
- **WHEN** 页面绘制目录或文件项
- **THEN** 目录显示 `[D]`，文件显示 `[F]`

## MODIFIED Requirements
### Requirement: Folders 列表展示与交互策略
原“单一默认排序 + 固定字体 + 常规滚动 + 长文件名单行截断 + `[FILE]/[DIR]` 标记”修改为“可选大小升序 + 三档字体 + 快速滑块 + 文件名两行限制 + `[F]/[D]` 标记”。
